# Troubleshooting

Find solutions to common problems encountered while using Kubetrace. If your issue is not listed, please contact support at [support@kubetrace.co](mailto:support@kubetrace.co).

## Error Code Reference

Below is a comprehensive list of error codes you may encounter in Kubetrace. Click on any error to expand and see the error message and detailed solutions.

<AccordionGroup>
  <Accordion title="401 (Unauthorized) - Login Failed | Authentication | Login">
    **Scenario:** Wrong password
    
    **Error Message:** Login failed
    
    **Description & Solution:**
    - Verify your login credentials are correct
    - Check for caps lock and ensure correct password capitalization
    - Reset your password if needed using the "Forgot Password" option
    - Clear browser cookies and try logging in again
  </Accordion>

  <Accordion title="400 (Bad Request) - Email Already Registered | Registration | Register">
    **Scenario:** Email already registered
    
    **Error Message:** Email already registered
    
    **Description & Solution:**
    - This email address is already associated with an existing account
    - Use the login page if you already have an account
    - Try the "Forgot Password" option to recover your account
    - Use a different email address if you want to create a new account
  </Accordion>

  <Accordion title="400 (Bad Request) - Invalid Email | Registration | Register">
    **Scenario:** Invalid / temporary email
    
    **Error Message:** Please use a valid, permanent email address to continue
    
    **Description & Solution:**
    - Temporary or disposable email addresses are not accepted
    - Use a corporate or personal email address from a recognized provider
    - Ensure the email format is correct (e.g., user@domain.com)
    - Check for typos in the email address
  </Accordion>

  <Accordion title="404 (Not Found) - Page Not Found | Routing | Common">
    **Scenario:** API / Page not found
    
    **Error Message:** Not Found
    
    **Description & Solution:**
    - The requested page or API endpoint does not exist
    - Check the URL for typos or incorrect paths
    - Navigate back to the dashboard and try accessing the page again
    - Clear browser cache and reload the page
    - Contact support if the page should exist but isn't accessible
  </Accordion>

  <Accordion title="404 (Not Found) - Subscription Restriction | Subscription | Access Control">
    **Scenario:** Subscription plan not allowed
    
    **Error Message:** Your subscription plan not allowed to Add cluster
    
    **Description & Solution:**
    - Your current subscription plan has reached the cluster limit
    - Review your current subscription plan limits in account settings
    - Upgrade your subscription to add more clusters
    - Remove unused clusters to free up capacity
    - Contact sales to discuss enterprise options
  </Accordion>

  <Accordion title="422 (Unprocessable Entity) - User Not Found | Team | Myteam">
    **Scenario:** Invitation for non-Kubetrace user
    
    **Error Message:** user not registered in kubetrace
    
    **Description & Solution:**
    - The invited user does not have a Kubetrace account
    - Ask the user to register for Kubetrace first at the registration page
    - Verify the email address is correct
    - Send them an invitation link to sign up
    - Once registered, resend the team invitation
  </Accordion>

  <Accordion title="500 (Internal Server Error) - Topology Failed | Topology | Topology main page">
    **Scenario:** If selected cluster disconnected, topology will fail
    
    **Error Message:** Failed to fetch topology
    
    **Description & Solution:**
    - The selected cluster is disconnected or unreachable
    - Verify the selected cluster is connected and healthy in cluster management
    - Check cluster status and ensure it's online
    - Ensure the cluster operator is running properly
    - Verify network connectivity between the cluster and Kubetrace service
    - Try selecting a different cluster to isolate the issue
    - Refresh the topology page after ensuring cluster connectivity
  </Accordion>

  <Accordion title="410 (Gone) - Add/Delete Cluster Unauthorized | Cluster | Cluster main page">
    **Scenario:** If user have executer permission and try to Add/Delete the team cluster
    
    **Error Message:** not authorized, your role is not authorized to Add/Delete this cluster
    
    **Description & Solution:**
    - Users with Executor role cannot add or delete clusters
    - Check your user role in team settings
    - Contact your team administrator to request Cluster-Manager permissions
    - Only users with Cluster-Manager or Admin roles can add/delete clusters
    - Review team permission documentation for role capabilities
  </Accordion>

  <Accordion title="410 (Gone) - Enable/Disable Cluster Unauthorized | Cluster | Cluster main page">
    **Scenario:** If user have executer permission and try to Enable/Disable the team cluster
    
    **Error Message:** not authorized, your role is not authorized to Enable/Disable this cluster
    
    **Description & Solution:**
    - Users with Executor role cannot enable or disable clusters
    - Verify your user role permissions in team settings
    - Request Cluster-Manager role from your team administrator
    - Only authorized roles can manage cluster state
    - Contact support if you need help with role assignments
  </Accordion>

  <Accordion title="410 (Gone) - Execute Test Run Unauthorized | Cluster | Cluster main page">
    **Scenario:** If user have Cluster-manager permission and try to execute test run the team cluster
    
    **Error Message:** not authorized, your role is not authorized to execute the test run from this cluster
    
    **Description & Solution:**
    - Users with Cluster-Manager role cannot execute test runs
    - Check your role - only Executor role can run tests
    - Request Executor permissions from your team administrator
    - Verify the role-based access control (RBAC) settings for your team
    - Review documentation on role separation between cluster management and test execution
  </Accordion>

  <Accordion title="410 (Gone) - Create Schedule Unauthorized | Cluster | Cluster main page">
    **Scenario:** If user have Cluster-manager permission and try to create schedule in the team cluster
    
    **Error Message:** not authorized, your role is not authorized to create the schedule from this cluster
    
    **Description & Solution:**
    - Users with Cluster-Manager role cannot create test schedules
    - Verify you have the appropriate role to create schedules
    - Contact your team administrator for Executor permissions
    - Review the permission requirements for schedule creation
    - Check team settings to understand role capabilities
  </Accordion>

  <Accordion title="504 (Gateway Timeout) - No Data Available | Cluster/Test Suites | Common">
    **Scenario:** If page is not loaded
    
    **Error Message:** No data available
    
    **Description & Solution:**
    - The page failed to load data within the timeout period
    - Refresh the page and try again
    - Check your internet connection
    - Verify the cluster is accessible and responding
    - Clear browser cache and reload
    - Wait a few moments and retry - there may be temporary connectivity issues
    - Contact support if the issue persists
  </Accordion>

  <Accordion title="403 (Forbidden) - Invalid Kubeconfig | Cluster | Add Cluster">
    **Scenario:** Wrong kubeconfig / invalid config
    
    **Error Message:** Please check your kubeconfig
    
    **Description & Solution:**
    - The provided kubeconfig file is invalid or contains incorrect credentials
    - Validate your kubeconfig file format is correct YAML
    - Ensure the cluster API server endpoint is accessible
    - Verify your credentials in the kubeconfig have not expired
    - Check for typos or missing fields in the configuration
    - Test the kubeconfig with kubectl locally to ensure it works
    - Regenerate the kubeconfig from your cluster if needed
  </Accordion>

  <Accordion title="409 (Conflict) - Cluster Already Added | Cluster | Add Cluster">

    - Check if the cluster is already registered in your cluster list
    - Review the cluster list to find the existing cluster
    - Use a different cluster name if you want to add it as a separate entry
    - Remove the existing cluster first if you want to re-add it with new credentials
    - Contact support if you believe this is an error
  </Accordion>

## Common Issues

### 1. Cannot Connect Cluster

**Symptoms:**
- Unable to establish connection to your Kubernetes cluster
- Connection timeout errors

**Solutions:**
- Ensure your kubeconfig file is correct and up to date
- Verify the cluster API server endpoint is accessible
- Check network connectivity between your machine and the Kubernetes API server
- Confirm that your firewall settings allow outbound connections to the cluster
- Validate that your credentials in the kubeconfig have not expired

---

### 2. Operator Deployment Fails

**Symptoms:**
- Operator pod fails to start
- Deployment errors in Kubernetes

**Solutions:**
- Verify you have the necessary permissions (RBAC) to deploy resources in the cluster
- Check for typos or missing fields in the YAML manifest file
- Ensure the namespace exists before deploying
- Review operator logs using `kubectl logs` for detailed error messages
- Confirm that required Custom Resource Definitions (CRDs) are installed

---

### 3. Test Results Not Showing

**Symptoms:**
- Dashboard shows no test results
- Test execution completes but data is not visible

**Solutions:**
- Confirm the operator is running and healthy: `kubectl get pods -n <namespace>`
- Refresh the dashboard and clear your browser cache
- Check operator logs for any errors: `kubectl logs <operator-pod> -n <namespace>`
- Verify that the test configuration is correct
- Ensure network connectivity between the operator and the Kubetrace backend

---

### 4. Authentication Issues

**Error Codes:** 401, 403

**Solutions:**
- Verify your login credentials are correct
- Clear browser cookies and try logging in again
- Reset your password if needed
- Contact support if the issue persists

---

### 5. Permission Denied Errors

**Error Codes:** 410, 403

**Solutions:**
- Check your user role and permissions in Kubetrace
- Contact your team administrator to request appropriate access
- Verify you're performing actions within your role's scope (executor vs cluster-manager)

---

### 6. Cluster Addition Problems

**Error Codes:** 403, 409, 500

**Solutions:**
- **Wrong kubeconfig (403):** Validate your kubeconfig file format and credentials
- **Cluster already added (409):** Check if the cluster is already registered in your account
- **Internal errors (500):** Wait a few moments and try again, or contact support

---

### 7. Topology View Issues

**Error Code:** 500

**Solutions:**
- Verify the selected cluster is connected and healthy
- Check if the cluster has network connectivity to the Kubetrace service
- Refresh the topology page
- Ensure the cluster operator is running properly

## Getting Additional Help

If you continue experiencing issues after trying these solutions:

1. **Check the Status Page:** Visit our status page to see if there are any known service interruptions
2. **Review Documentation:** Consult our comprehensive documentation at [docs.kubetrace.co](https://docs.kubetrace.co)
3. **Contact Support:** Email us at [support@kubetrace.co](mailto:support@kubetrace.co) with:
   - Description of the issue
   - Error code (if applicable)
   - Steps to reproduce
   - Screenshots (if relevant)
   - Your cluster and Kubetrace version information

---

## Preventive Measures

To minimize issues:

- Keep your Kubetrace operator updated to the latest version
- Regularly review and update your kubeconfig files
- Monitor cluster health and resource usage
- Set up proper RBAC policies
- Review operator logs periodically
- Maintain good network connectivity between clusters and Kubetrace services
